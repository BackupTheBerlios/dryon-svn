Natives

	Functions useable in the scripts
	all the maxsize parameters have a default value set
	you don't need to specify it youself, just use the default value
	It is the length of the array (not the max length of the string in it !), so if you declare
	an array like this:  new str[20];
	20 is the size to be given to the function as maxsize for str
	to use the default value for a parameter which is not the last one use '_'
	def: native myfunc(a= 2, b);
	use: myfunc( _, 5);

TimeDateFuncs

getCurrentDate ( since 0.1 )

USAGE:

	date:getCurrentDate(bool:with_time= true)

RETURN VALUE:

	return timestamp for current datetime

INPUTS:

	with_time : if false the HH:MM:SS part is set to 00:00:00

getDateDiff ( since 0.1 )

USAGE:

	getDateDiff(date:timestamp1, date:timestamp2, type)

RETURN VALUE:

	integer : difference between the two timestamp

INPUTS:

	timestamp1 : timestamp for the newer date
	timestamp2 : timestamp for the older date
	type       : return type
				 's' = return diff in seconds
			     'm' = return diff in minutes
			     'h' = return diff in hours
			     'd' = return diff in days

makeDate ( since 0.1 )

USAGE:

	date:makeDate(const datestr[], bool:parse_time= true)

INPUTS:

	datestr    : datetime in MySQL format (YYY-MM-DD HH:MM:SS)
	parse_time : if false the HH:MM:SS is ignored in datestr and
				 00:00:00 is used instead to generate the timestamp

DESCRIPTION:

	build a timestamp from  date in MySQL format

ScriptCallbacks

	functions called by the engine when events are triggered
	bot is only aware of the events happening in a channel where it is
	bot can't receive a join event for user Bob on channel #fgh if it
	is not on it.
	Its the basic of IRC but i felt like i had to put it there in case.

event_LoadPlugin ( since 0.3 )

USAGE:

	public event_LoadPlugin()

DESCRIPTION:

	Called when plugin is loaded by the engine, you can't
	be sure that the bot is connected to the IRC server there
	so don't call any irc_* function here

USAGE:

DESCRIPTION:

	Called when plugin is unloaded by the engine, you can't
	be sure that the bot is connected to the IRC server there
	so don't call any irc_* function here

FieldConversions

	used to retrieve field data

mysql_get_DATEfield

USAGE:

	mysql_get_DATEfield(mysql_res:result_index, array_index, const field_name[], buff[], buff_maxsize= sizeof buff, const dateformat[])

INPUTS:

	result_index : result identifier returned by mysql_query
	array_index  : result line ( use mysql_count_results to get the line count )
	field_name   : field name ( case sensitive )
	buff         : where to store the result
	buff_maxsize : Natives
	dateformat   : format string ( will be passed as is to strftime system call
					Here is a non-exhaustive list of the available options:
					%d = day of the month ("01"-"31")
					%e = day of the month (" 1"-"31")
					%m = month ("01"-"12")
					%F = "%Y-%m-%d"
					%H = hour ("00"-"23") 24h
					%I = hour ("01"-"12") 12h
					%M = minutes ("00"-"59")
					%S = seconds ("00"-"60")
					%R = "%H:%M"
					%T = "%H:%M:%S"
					%n = newline
					%t = tabulation

DESCRIPTION:

	used to retrieve value of a result field as a formatted date string (DATETIME fields)

mysql_get_INTfield

USAGE:

	mysql_get_INTfield(mysql_res:result_index, array_index, const field_name[])

RETURN VALUE:

	integer : field data converted to integer

INPUTS:

	result_index : result identifier returned by mysql_query
	array_index  : result line ( use mysql_count_results to get the line count )
	field_name   : field name ( case sensitive )

DESCRIPTION:

	used to retrieve value of a result field as an integer
